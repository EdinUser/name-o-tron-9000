# Rename Files Using Plex Metadata

Name-o-Tron renames files using metadata already matched in Plex. It does not ask you to identify media again through a separate scraper. Plex remains the source of truth; Name-o-Tron applies that resolved identity to your filesystem.

This is useful when your files play correctly in Plex but the underlying names and folders are inconsistent, hard to move, or not self-explanatory.

## Before and After

```text
Bad.Movie.Name.1080p.x264.mkv
Bad.Movie.Name.1080p.x264.eng.srt
```

can become:

```text
Blade Runner (1982) {imdb-tt0083658}/
├── Blade Runner (1982) {imdb-tt0083658}.mkv
└── Blade Runner (1982) {imdb-tt0083658}.eng.srt
```

For TV episodes:

```text
show.s01e01-e02.web.mkv
```

can become:

```text
Show Name/
└── Season 01/
    └── Show Name - S01E01-E02 - Episode Title.mkv
```

The exact output depends on your templates and settings.

## Workflow

1. Connect to Plex and choose a library.
2. Map Plex server paths to local paths if the server and app see storage differently.
3. Choose a template for Movies, TV Shows, or Music.
4. Preview the proposed names and folders.
5. Review green, yellow, red, and unmatched statuses.
6. Apply only selected safe operations.
7. Use operation logs or rollback if a completed batch needs to be reversed.

## Why Use Plex Metadata?

Plex already knows which movie, episode, album, or track each item represents. Name-o-Tron uses that curated match instead of guessing from filenames.

That avoids a common failure mode of general-purpose renamers: a second identification pass that disagrees with the media server you already trust.

## What Can Be Renamed

- Movie files and movie folders.
- TV episodes, season folders, specials, and multi-episode files.
- Music tracks, album folders, disc folders, and artist layouts.
- Subtitle sidecar files associated with media.
- Related structures generated by collection, year, genre, alphabetical, or template settings.

## Safety Checks

Name-o-Tron validates operations before applying them:

- target already exists
- duplicate targets in the same batch
- invalid characters
- reserved names
- path length risks
- permission problems
- unsupported or uncertain file operations

Red items must be fixed or skipped before proceeding. Yellow items are warnings that remain visible for review.

For the full safety model, see [Features](features.md#preview-safety-system). For template configuration, see [Renaming & Templates](renaming-and-templates.md).
